# СП-325 Клокун, Рабин

nbc-main --- это имплементация наивного байесовского классификатора на Python с использованием средств стандартной библиотеки.

Как пользоваться:
1. Если Python 3 установлен, находится в $PATH и вы под Linux/macOS/POSIX-совместимой ОС:
1.1. Открыть терминал
1.2. Перейти в папку со скриптом nbc-main.py
1.3. Запустить так ('$' не вводится, а только обозначает текущий рядок, в который вводится команда, и выполнение от имени текущего пользователя)
$ chmod +x nbc-main.py
$ ./nbc-main.py <dataset>
Например:
$ ./nbc-main.py iris.csv
1.4. Наблюдать результат, например:

$ ./nbc-main.py iris.csv
Entry: "[7.7, 3.8, 6.7, 2.2, 3.0]", c_act: "3.0", c_pred: "3.0"
Entry: "[5.8, 2.8, 5.1, 2.4, 3.0]", c_act: "3.0", c_pred: "3.0"
...
Entry: "[5.6, 3.0, 4.1, 1.3, 2.0]", c_act: "2.0", c_pred: "2.0"
Entry: "[6.4, 3.1, 5.5, 1.8, 3.0]", c_act: "3.0", c_pred: "3.0"
Accuracy: 98.0

2. Если Python 3 установлен, находится в %PATH% и вы под Windows:
2.1. Открыть терминал
2.2. Перейти в папку со скриптом nbc-main.py
2.3. Запустить так ('>' не вводится, а только обозначает текущий рядок, в который вводится команда, и выполнение от имени текущего пользователя)
> python nbc-main.py <dataset>
Например:
> python nbc-main.py iris.csv
2.4. Наблюдать результат, например:
> python nbc-main.py iris.csv
Entry: "[7.7, 3.8, 6.7, 2.2, 3.0]", c_act: "3.0", c_pred: "3.0"
Entry: "[5.8, 2.8, 5.1, 2.4, 3.0]", c_act: "3.0", c_pred: "3.0"
...
Entry: "[5.6, 3.0, 4.1, 1.3, 2.0]", c_act: "2.0", c_pred: "2.0"
Entry: "[6.4, 3.1, 5.5, 1.8, 3.0]", c_act: "3.0", c_pred: "3.0"
Accuracy: 98.0

3. Если Python 3 установлен или не находится в %PATH% и вы под Windows:
3.1. Открыть командную строку.
3.2. Перейти в папку с исполняемым файлом nbc-main.exe, а именно: dist/nbc-main/nbc-main.exe
3.3. Запустить так ('>' не вводится, а только обозначает текущий рядок):
> nbc-main.exe ..\..\<dataset>
Например:
> nbc-main.exe iris.csv
3.4. Наблюдать результат, например:
>nbc-main.exe ..\..\iris.csv
Entry: "[6.7, 3.1, 4.7, 1.5, 2.0]", c_act: "2.0", c_pred: "2.0"
Entry: "[6.2, 2.9, 4.3, 1.3, 2.0]", c_act: "2.0", c_pred: "2.0"
...
Entry: "[5.4, 3.7, 1.5, 0.2, 1.0]", c_act: "1.0", c_pred: "1.0"
Entry: "[7.0, 3.2, 4.7, 1.4, 2.0]", c_act: "2.0", c_pred: "2.0"
Accuracy: 98.0
